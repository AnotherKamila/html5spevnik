Hello,
I am that thing's project page.

That thing is a neat, touch-friendly, and human-friendly songbook/sheet music organizer written as an HTML5 offline app.

The Plan

It should have an almost paper-like feeling, at least in being minimal - no distractions.

It should be easy to use for me and for my mother.

It should work offline.

It should work across all devices with modern browsers.

It is not planned to be in a working state anytime soon.

Extensible!

Even supposedly core parts (like displaying text/sheet music/guitar tabs) will actually be a form of plugins. Therefore, it might be possible to use it for more purposes than just keeping my sheet music tidy.

Why all this

The main reason why I am doing this is that I want to (1) try to write a semi-serious web app, (2) learn about all the new html5 awesomeness and (3) get the impression that I know Javascript.

I cannot avoid mentioning my long-term frustration about never having coded anything interesting. I do a lot of algorithm design competitions, which is cool, but that does not really give me the experience needed in The Real World. I work for a web design company, but that's just struggling with CSS/swearing about JQuery Mobile most of the time, and that is not even a bit cool. So this is my first attempt at actually doing some coding that I might (possibly, hopefully) end up maintaining and not hating at the same time.
